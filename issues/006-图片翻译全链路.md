# #6 图片翻译全链路：Excel输入 → OCR→翻译→擦除→覆写→改尺寸→R2上传 → Excel+报告输出

**状态**：待开始 | **类型**：AFK | **阻塞于**：#1, #2, #3 | **GitHub**：[#6](https://github.com/linhuanwen/wb-content-tool/issues/6)

## What to build

实现图片翻译 Tab 的完整功能，输入含 `图片url` 列的 Excel（兼容 4 列或 12 列），对每张图片执行全流程本地化处理，输出 URL 全替换为 R2 地址的 Excel + 详细处理报告。

端到端行为：
1. 用户上传 Excel（兼容爬虫 4 列或翻译后 12 列），系统自动识别 ASIN 和图片 URL 列表
2. 用户可选 1 个 ASIN 进行试跑预览——并排展示原图 vs 俄语图，可调整字体配置
3. 确认效果后启动全量处理
4. 每个 ASIN 的多张图并发处理，ASIN 间串行（支持断点续跑 progress.json）
5. 单张图处理管线：下载原图 → PaddleOCR 检测定位 → DeepSeek 翻译文字 → AI 修复擦除 → Pillow 覆写俄文 → 改 3:4（900×1200）→ 上传 R2 + 本地存档
6. 无文字图片跳过翻译步骤，只改尺寸 + 上传
7. 任意环节失败不阻塞同 ASIN 其他图（失败图兜底：改尺寸上传不写字）
8. 输出 Excel 中图片 URL 全部替换为 R2 URL
9. 额外输出详细处理报告（每张图状态：成功/跳过/失败 + 错误原因）
10. 侧边栏提供 R2 凭证、图片尺寸模式、字体参数配置

## Acceptance criteria

- [ ] Tab 3「🖼️ 图片翻译」正常显示，可上传 Excel
- [ ] 兼容 4 列（爬虫输出）和 12 列（翻译输出）两种格式
- [ ] 试跑预览：选 1 个 ASIN，并排展示原图与俄语图对比
- [ ] PaddleOCR 正确检测文字区域并输出坐标
- [ ] 有文字图片：完整走 OCR→翻译→擦除→覆写→改尺寸→上传
- [ ] 无文字图片：仅改尺寸 + 上传（不翻译）
- [ ] 输出图片分辨率均为 900×1200（3:4）
- [ ] 俄文覆写位置与原英文位置一致
- [ ] 本地存档按 `images/{ASIN}/{序号}_{标记}.jpg` 组织
- [ ] R2 上传成功后返回公开可访问的 URL
- [ ] 输出 Excel 的 图片url 列全部替换为 R2 URL
- [ ] 处理报告单独可下载，包含每张图的处理状态
- [ ] 单张图失败不影响同 ASIN 其他图
- [ ] 断点续跑：中断后重启从上次进度继续
- [ ] 进度条实时显示整体进度
- [ ] 不配置 R2 凭证时给出明确提示，不崩溃
- [ ] 不配置翻译 API 时给出明确提示，不崩溃
