# #1 项目骨架搭建：配置管理 + Prompt + 依赖

**状态**：待开始 | **类型**：AFK | **阻塞**：无 | **GitHub**：[#1](https://github.com/linhuanwen/wb-content-tool/issues/1)

## What to build

搭建项目基础设施，让后续所有模块有统一的配置入口和翻译 Prompt。

- **config.py**：通过 `.env` 文件管理 `TRANSLATE_API_PROVIDER`、`TRANSLATE_API_KEY`、`TRANSLATE_API_BASE_URL`、`TRANSLATE_MODEL`、`CRAWLER_HEADLESS`、`CRAWLER_DELAY` 等配置项
- **prompts/translation_persona.txt**：从现有"文案转化AI人设.docx"提取完整内容作为 System Prompt，末尾追加 JSON 格式输出指令
- **.env.example**：提供配置模板供用户复制
- **requirements.txt**：列出所有 Python 依赖

## Acceptance criteria

- [ ] .env.example 包含所有配置项的模板和说明
- [ ] config.py 能从 .env 读取所有配置，提供合理的默认值
- [ ] prompts/translation_persona.txt 包含完整人设内容 + JSON 输出格式要求
- [ ] requirements.txt 列出 streamlit、playwright、openpyxl、pandas、openai、anthropic、python-dotenv、pydantic
- [ ] 无需安装任何包即可检查以上文件内容
