# #3 AI翻译全链路：处理前表格输入 → AI翻译 → 处理后表格输出

**状态**：待开始 | **类型**：AFK | **阻塞于**：#1 | **GitHub**：[#3](https://github.com/linhuanwen/wb-content-tool/issues/3)

## What to build

实现 AI 翻译模块，输入"爬虫表格（处理前）"，调用 AI API 生成俄语 SEO 优化文案，输出"爬虫表格（处理后）"。

端到端行为：
1. 读取"处理前"Excel（含 asin, 图片url, 标题, 详情）
2. 定义抽象接口 `TranslationProvider`（ABC），内置 `OpenAICompatibleProvider` 和 `ClaudeProvider` 两个实现
3. 工厂函数根据 `TRANSLATE_API_PROVIDER` 配置选择实例
4. 使用 `prompts/translation_persona.txt` 作为 System Prompt
5. 将英文标题+详情发送给 AI，要求返回 JSON：`{"核心流量词":"...","俄语标题":"...","俄语详情":"..."}`
6. 后处理：俄语标题 ≤ 60 字符、全小写、无特殊符号
7. JSON 解析失败自动重试一次
8. 输出 Excel 包含 12 列：asin, 图片url, 标题, 详情, 核心流量词, 俄语标题, 俄语详情, 货源(空), 采购价(空), (空), (空), 商品类别(空)
9. 提供 CLI 入口：`python translator.py --input 处理前.xlsx --output 处理后.xlsx`
10. 提供 `def translate_batch(products, progress_callback=None)` 函数供 Web 界面调用

## Acceptance criteria

- [ ] 用"处理前"样本数据测试，能返回结构正确的俄语文案
- [ ] 俄语标题 ≤ 60 字符，全小写，无特殊符号
- [ ] 处理后的核心流量词、俄语标题、俄语详情均非空
- [ ] 输出表格结构与"爬虫表格（处理后）"样本一致（12 列）
- [ ] 货源、采购价、商品类别三列为空
- [ ] CLI 命令可正常运行并输出 xlsx 文件
- [ ] 不配置 API Key 时给出明确提示，不崩溃
