# #5 Web界面 — 文案翻译功能区（Streamlit Tab2）+ API配置

**状态**：待开始 | **类型**：AFK | **阻塞于**：#3 | **GitHub**：[#5](https://github.com/linhuanwen/wb-content-tool/issues/5)

## What to build

在 Streamlit Web 界面上实现"功能区2：文案翻译"Tab 和侧边栏 API 配置。

端到端行为：
1. 用户选择"文案翻译" Tab
2. 上传"处理前"Excel 文件
3. 系统校验必须包含 [asin, 图片url, 标题, 详情] 列
4. 显示"检测到 X 个待翻译产品"
5. 点击"开始翻译"按钮
6. 实时进度条显示翻译进度
7. 翻译完成后在页面上以表格预览结果
8. 点击"下载处理后表格"按钮，导出为标准 12 列 xlsx
9. 侧边栏提供 API 配置界面：Provider 下拉选择、API Key 输入框、API Base URL 输入框、模型名称输入框
10. 侧边栏配置可保存到本地 .env 文件
11. 翻译失败的产品在日志区域显示警告

## Acceptance criteria

- [ ] 上传非 Excel 文件时给出友好提示
- [ ] 上传 Excel 但缺少必要列时给出提示
- [ ] 能成功翻译至少 2 条产品并在页面预览
- [ ] 下载的 xlsx 文件与"爬虫表格（处理后）"格式一致（12 列，空列保留）
- [ ] 侧边栏 API 配置能保存并在下次打开时自动加载
- [ ] 未配置 API Key 时点击翻译给出提示，不崩溃
- [ ] 进度条实时更新，不卡死 UI
