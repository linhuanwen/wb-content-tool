# 两阶段 AI 翻译流水线替代单阶段翻译

用 Phase 1（信息萃取）+ Phase 2（文案生成）两阶段流水线替代原有的单次 AI 翻译调用。Phase 1 将产品页 HTML 拆解为结构化属性记录，Phase 2 基于完整属性表生成俄语文案，而非仅凭标题+详情两段文本"猜测"。

## 为什么

- **源信息利用**：单阶段方案只传递标题和详情给 AI，丢失了大量隐藏在 HTML 中的产品属性（材质、规格、品牌等）。两阶段方案让 AI 先读完整页面再动笔。
- **可复用性**：结构化数据库建完后，换平台（Ozon 等）或换 persona 只需重跑 Phase 2，不重复萃取。
- **去品牌化精度**：Phase 1 显式提取品牌名，Phase 2 可精准替换，避免当前 prompt 中"笼统要求去品牌"导致的遗漏或误伤。

## 考虑的替代方案

- **保留 extractor.py + 仅增强 translator 的 prompt**：改动最小，但不解决信息丢失问题。extractor 的 CSS selector 解析永远无法提取"目标用户"、"使用场景"等语义信息。
- **将 HTML 直接喂给 translator**：省掉了 Phase 1，但单次 prompt 同时要求信息萃取 + 文案创作，指令冲突且 token 极高。
